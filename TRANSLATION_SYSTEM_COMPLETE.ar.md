# نظام الترجمة - التنفيذ الكامل

## نظرة عامة

يدعم نظام بومبا الآن **146 لغة عالمية** مع نظام احتياطي ذكي. يمكن للمستخدمين اختيار أي لغة من صفحة الإعدادات، وسيتعامل النظام تلقائيًا مع الترجمات.

## تفاصيل التنفيذ

### 1. دعم اللغات

**إجمالي اللغات**: 146 لغة من جميع أنحاء العالم

**الترجمات الكاملة المتاحة**:
- 🇸🇦 العربية (Arabic) - `ar.json` - 3872 سطر
- 🇬🇧 الإنجليزية (English) - `en.json` - 3876 سطر
- 🇫🇷 الفرنسية (Français) - `fr.json` - 3872 سطر

**الترجمات الأساسية**: 143 لغة مع المفاتيح الأساسية
- جميع اللغات الأخرى لديها ملفات ترجمة أساسية
- المفاتيح المفقودة تعود تلقائيًا إلى الإنجليزية
- كل ملف يتضمن بيانات وصفية للغة

### 3. كيف يعمل

#### اختيار اللغة
1. المستخدم يفتح صفحة الإعدادات
2. يرى قائمة منسدلة بجميع الـ 146 لغة بأسمائها الأصلية
3. يختار اللغة المطلوبة (مثلاً: "中文 (Chinese)")
4. النظام يحدث الواجهة فورًا

#### سلسلة الاحتياط للترجمة
```
المستخدم يختار الصينية (zh) → 
  البحث في zh.json عن المفتاح → 
    إذا وُجد: استخدام الترجمة الصينية
    إذا لم يُوجد: العودة إلى الإنجليزية (en.json)
```

#### دعم RTL
- 9 لغات لديها دعم RTL (من اليمين إلى اليسار):
  - العربية (ar)، العبرية (he)، الفارسية (fa)، الأردية (ur)
  - البشتو (ps)، السندية (sd)، الأويغورية (ug)، اليديشية (yi)، الديفيهي (dv)
- النظام يبدل اتجاه النص تلقائيًا
- التخطيط يتكيف مع لغات RTL

### 5. دعم الخادم (Backend)

**نماذج قاعدة البيانات المحدثة**:
- `User.js` - حقل اللغة يقبل أي كود ISO 639-1/639-2 صالح (2-3 أحرف)
- `Organization.js` - حقل اللغة يقبل أي كود لغة صالح
- التحقق يستخدم regex: `/^[a-z]{2,3}$/`

**نقاط النهاية API**:
- `PUT /api/settings/general` - يقبل أي كود لغة
- لا مزيد من قيود enum
- يتحقق من التنسيق فقط (2-3 أحرف صغيرة)

### 6. السكريبتات المتاحة

#### إنشاء جميع الترجمات
```bash
node src/i18n/generate-all-translations.js
```
- ينشئ ملفات ترجمة أساسية لجميع الـ 146 لغة
- يتخطى اللغات التي لديها ترجمات بالفعل (ar, en, fr)
- كل ملف ~50 سطر مع المفاتيح الأساسية

#### إنشاء قالب ترجمة
```bash
node src/i18n/create-translation-template.js
```
- يستخرج جميع مفاتيح الترجمة من ar.json
- ينشئ قالب لإضافة ترجمات كاملة جديدة
- مفيد للمترجمين

### 7. إضافة ترجمات كاملة

لإضافة ترجمات كاملة للغة:

1. **ابحث عن ملف اللغة**: `src/i18n/locales/[code].json`
2. **استخدم الإنجليزية كمرجع**: انسخ الهيكل من `en.json`
3. **ترجم جميع المفاتيح**: استبدل النص الإنجليزي باللغة المستهدفة
4. **حدّث البيانات الوصفية**: غيّر `translationStatus` إلى `"full"`
5. **اختبر**: اختر اللغة في الإعدادات وتحقق

مثال للإسبانية (es):
```bash
# انسخ قالب الإنجليزية
cp src/i18n/locales/en.json src/i18n/locales/es.json

# عدّل es.json وترجم جميع القيم
# احتفظ بجميع المفاتيح كما هي، ترجم القيم فقط
```

### 9. تجربة المستخدم

**للمستخدمين**:
- اختيار أي من 146 لغة في الإعدادات
- تحديث الواجهة فورًا (بدون إعادة تحميل الصفحة)
- عرض أسماء اللغات الأصلية
- انتقالات سلسة بين RTL/LTR

**للمديرين**:
- سهولة إضافة ترجمات كاملة جديدة
- نظام الاحتياط يمنع كسر الواجهة
- السكريبتات تؤتمت إنشاء ملفات الترجمة

**للمطورين**:
- تكامل i18n بسيط: `t('key.path')`
- دعم TypeScript لمفاتيح الترجمة
- اكتشاف اللغة تلقائيًا
- إعدادات اللغة مركزية

### 10. أفضل الممارسات

**عند إضافة الترجمات**:
1. استخدم دائمًا نفس هيكل المفاتيح كالإنجليزية
2. احتفظ بالمتغيرات كما هي: `{{variable}}`
3. احتفظ بوسوم HTML إن وُجدت
4. اختبر لغات RTL بدقة
5. حدّث البيانات الوصفية في ملف الترجمة

**عند استخدام الترجمات في الكود**:
```typescript
// جيد
const text = t('common.save');

// جيد مع المتغيرات
const message = t('auth.welcome', { name: userName });

// جيد مع احتياطي
const text = t('new.key', 'Default text');
```

### 11. الاختبار

**الاختبار اليدوي**:
1. افتح صفحة الإعدادات
2. غيّر اللغة إلى أي من 146 خيار
3. تنقل عبر جميع الصفحات
4. تحقق من عرض النص بشكل صحيح
5. تحقق من لغات RTL لمشاكل التخطيط

**الاختبار الآلي**:
```bash
# تشغيل اختبارات الواجهة الأمامية
npm test

# التحقق من مفاتيح الترجمة المفقودة
node src/i18n/create-translation-template.js
```

### 12. استكشاف الأخطاء

**المشكلة**: اللغة لا تظهر في القائمة المنسدلة
- **الحل**: تحقق من أن `shared/languages.js` يتضمن اللغة

**المشكلة**: مفتاح الترجمة غير موجود
- **الحل**: المفتاح سيعود إلى الإنجليزية تلقائيًا

**المشكلة**: تخطيط RTL معطل
- **الحل**: تحقق من أن اللغة محددة كـ RTL في `shared/languages.js`

**المشكلة**: الخادم يرفض كود اللغة
- **الحل**: تأكد من أن الكود 2-3 أحرف صغيرة (تنسيق ISO 639)
