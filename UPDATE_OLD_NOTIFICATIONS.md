# تحديث الإشعارات القديمة لدعم اللغات المتعددة

## المشكلة
الإشعارات القديمة في قاعدة البيانات تم إنشاؤها قبل إضافة دعم اللغات المتعددة، لذا لا تحتوي على `metadata.translations`. هذا يعني أنها تظهر بلغة واحدة فقط (عادة العربية) ولا تتغير عند تغيير اللغة.

## الحل
تم إنشاء سكريبت `update-old-notifications.js` لتحديث جميع الإشعارات القديمة وإضافة الترجمات الثلاث (العربية، الإنجليزية، الفرنسية) إلى `metadata.translations`.

## كيفية التشغيل

### الخطوة 1: التأكد من الاتصال بقاعدة البيانات
تأكد من أن ملف `.env` في مجلد `server` يحتوي على:
```
MONGODB_URI=your_mongodb_connection_string
```

### الخطوة 2: تشغيل السكريبت
من مجلد `server`، قم بتشغيل:
```bash
npm run update:notifications
```

أو مباشرة:
```bash
node scripts/update-old-notifications.js
```

## ما يفعله السكريبت

1. **الاتصال بقاعدة البيانات**: يتصل بـ MongoDB باستخدام `MONGODB_URI`

2. **البحث عن الإشعارات القديمة**: يبحث عن جميع الإشعارات التي لا تحتوي على `metadata.translations`

3. **توليد الترجمات**: لكل إشعار:
   - يحاول تحديد نوع الإشعار من العنوان والرسالة
   - يولد الترجمات الثلاث باستخدام `notificationTranslations.js`
   - إذا فشل في تحديد النوع، يستخدم النص الأصلي للغات الثلاث

4. **تحديث قاعدة البيانات**: يحفظ الترجمات في `metadata.translations`

5. **عرض التقرير**: يعرض ملخصاً بعدد الإشعارات المحدثة والفاشلة

## مثال على الإخراج

```
✅ Connected to MongoDB
🔄 Starting notification update...
📊 Found 16497 notifications to update
✅ Updated 100 notifications...
✅ Updated 200 notifications...
...
✅ Updated 16497 notifications...

📈 Update Summary:
✅ Successfully updated: 16497
❌ Failed: 0
📊 Total: 16497
👋 Disconnected from MongoDB
```

## أنواع الإشعارات المدعومة

السكريبت يدعم توليد ترجمات تلقائية لـ:

### 1. إشعارات الجلسات (Session)
- جلسة جديدة (started)
- انتهاء الجلسة (ended)

### 2. إشعارات الطلبات (Order)
- طلب جديد (created)
- طلب جاهز (ready)

### 3. إشعارات المخزون (Inventory)
- مخزون منخفض (low_stock)

### 4. إشعارات الفواتير (Billing)
- دفع مكتمل (paid)
- دفع جزئي (partial_payment)

## ملاحظات مهمة

1. **النسخ الاحتياطي**: يُنصح بعمل نسخة احتياطية من قاعدة البيانات قبل تشغيل السكريبت:
   ```bash
   npm run backup
   ```

2. **الإشعارات الجديدة**: بعد تشغيل السكريبت، جميع الإشعارات الجديدة ستُنشأ تلقائياً بالترجمات الثلاث

3. **الإشعارات غير المعروفة**: إذا لم يتمكن السكريبت من تحديد نوع الإشعار، سيستخدم النص الأصلي للغات الثلاث

4. **التشغيل المتكرر**: يمكن تشغيل السكريبت عدة مرات بأمان - سيتخطى الإشعارات التي تحتوي بالفعل على ترجمات

## استكشاف الأخطاء

### خطأ في الاتصال بقاعدة البيانات
```
❌ MongoDB connection error: ...
```
**الحل**: تحقق من `MONGODB_URI` في ملف `.env`

### فشل في تحديث بعض الإشعارات
```
❌ Failed to update notification 123abc: ...
```
**الحل**: تحقق من سجلات الأخطاء وتأكد من صحة بيانات الإشعار

## التحقق من النتائج

بعد تشغيل السكريبت، يمكنك التحقق من النتائج:

1. افتح صفحة الإشعارات في التطبيق
2. غير اللغة من الإعدادات
3. تحقق من أن الإشعارات تتغير لغتها

## الملفات ذات الصلة

- `server/scripts/update-old-notifications.js` - السكريبت الرئيسي
- `server/utils/notificationTranslations.js` - ملف الترجمات
- `server/models/Notification.js` - موديل الإشعارات
- `src/pages/NotificationManagement.tsx` - صفحة عرض الإشعارات
- `src/components/NotificationCenter.tsx` - مركز الإشعارات
